## 🤝 介绍

WakePro 是 AstrBot 的高阶唤醒插件，把「能否唤醒」拆成一套固定顺序的处理流水线。
当前实现中，流水线顺序已经写死，**不再支持通过配置调整阶梯顺序**。

固定顺序如下：

| 顺序        | 配置键名                                         | 一句话职责                                             | 说明                                       |
| ----------- | ------------------------------------------------ | ------------------------------------------------------ | ------------------------------------------ |
| 前置过滤    | `global_blacklist` `blacklist` `whitelist` | 先决定消息是否需要继续进入唤醒判断                     | 这是进入流水线前的预处理，不属于可编排阶段 |
| ① 消息防抖 | `debounce`                                     | 命中已有防抖窗口时，优先合并同一用户的连续消息         | 只处理已开启的窗口                         |
| ② 阻塞判断 | `block`                                        | 判断当前这条消息是否应直接拒绝唤醒                     | 白名单用户可跳过                           |
| ③ 智能唤醒 | `wake`                                         | 真正决定“这句消息值不值得回”，并在成功后开启防抖监听 | 核心唤醒决策阶段                           |
| ④ 指令屏蔽 | `command`                                      | 避免把内置指令或前缀指令当成聊天                       | 防抖续发消息会跳过本阶段                   |
| ⑤ 沉默检测 | `silence`                                      | 唤醒后根据负面信号进入沉默期                           | 白名单用户可跳过                           |

---

### 前置过滤（名单与全局屏蔽）

“大门保安”——先过滤掉机器人自己、QQ 官方机器人、黑名单用户/群，或只允许白名单通过。
检查顺序：屏蔽自身 → 屏蔽 QQ 机器人 → 用户白名单 → 群白名单 → 用户黑名单 → 群黑名单。
（任何一步命中就直接放行或拦截，不再往下走。）

### ① 消息防抖（debounce）

“合并输入”——如果当前用户已经处在一个已开启的防抖窗口内，那么这一步会优先尝试把新消息合并进上一条尚未输出的请求。支持配置监听时长、最多合并条数，以及参与防抖的消息类型（`normal`、`at`、`reply`、`command`）。

- 命中防抖窗口的后续消息会沿用已唤醒状态，不再重新走完整判断。
- 检测到 GIF 时不会参与合并。
- 达到最大合并条数后，会立即发起请求，不再继续监听后续消息。

### ② 阻塞判断（block）

“红绿灯”——如果消息里含屏蔽词、在唤醒 CD 内、或是复读 Bot 上一条消息，则直接拒绝唤醒。

- 唤醒 CD 默认 0.5 s，每人独立计时，优先级最高。
- 屏蔽复读默认开启，可防止“复读机”把 Bot 无限叫起。

### ③ 智能唤醒（wake）

“大脑决策”——所有前置关卡通过后，才真正判断“这句消息值不值得回”。可叠加 7 种信号，全部独立开关、阈值可调：

1. 提及唤醒（@昵称）
2. 相关性唤醒（与 Bot 最近 5 条消息做语义相似度）
3. 答疑唤醒（检测到专业问句）
4. 兴趣唤醒（自定义关键词包）
5. 无聊唤醒（群聊低活跃）
6. 唤醒延长（被唤醒后 N 秒内持续在线）
7. 概率兜底（上面都没命中时，按固定概率随缘回）

当这一阶段判定为“需要回复”后，插件才会为该用户开启一段消息防抖监听窗口，后续连续消息再回到前面的 `debounce` 阶段进行合并。

### ④ 指令屏蔽（command）

“防误触”——把可能跟 AstrBot 内置指令冲突的消息提前拦掉。支持三种模式：

- 屏蔽内置指令（如 /llm /tts …）
- 屏蔽前缀指令（如 !help 不再触发插件指令）
- 屏蔽前缀 LLM（如 !聊天 不再唤醒大模型）

### ⑤ 沉默检测（silence）

“自我保护”——**仅在唤醒后生效**。
检测到“闭嘴、辱骂、人机”三类负面信号时，Bot 对该用户/群进入一段沉默时间，期间不再响应任何唤醒。
沉默时长 = 触发阈值 × 倍数，阈值越低“玻璃心”越重。

---

### 附：Bot 消息缓存

- Bot 消息缓存：始终保留最近 5 条 Bot 发言，用于相关性计算与复读屏蔽。
