# 更新日志

## v2.2.2

优化：

    优化管道配置，支持自定义启用的步骤、黑白名单作用的步骤

## v2.2.1

新功能：

- 当尚未配置任何关键词时，从内置的 `block_words.json` 文件中自动填充屏蔽关键词。

增强：

- 优化防抖处理逻辑，在存在 GIF 图片时跳过消息合并和窗口激活。
- 明确并重构 README 文档，以描述新的固定唤醒流水线阶段及其行为。
- 更新插件元数据描述，以反映修订后的唤醒流水线顺序。

文档：

- 重写 README 中的流水线章节，用固定、有序的唤醒流水线来描述各阶段的职责细节，包括防抖及机器人消息缓存。

## v2.2.0

新功能：

- 在 wake 流水线中添加一个 DebounceStep，将同一用户在可配置时间窗口内的连续消息合并为单个 wake 请求进行处理。
- 引入按用户跟踪待处理的 wake 请求，并带有自动过期和清理机制，以支持消息防抖行为。
- 在插件配置中新增可配置的防抖设置（窗口时长、最大合并数量以及适用的消息类型）。

增强：

- 将防抖阶段集成到现有的 wake 流水线中，使经过防抖的后续消息可以跳过 wake 冷却检查并复用之前的 wake 状态。
- 确保在即将发送消息时清除待处理的防抖请求，避免产生陈旧状态。
- 优化 README 文档，清晰描述新的消息防抖行为及其可配置性。
- 更新插件元数据描述，并将版本号提升至 v2.2.0，以反映新增的防抖阶段。
